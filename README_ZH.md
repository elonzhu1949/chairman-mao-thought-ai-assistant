## 🏗️ 6层知识体系架构

### 1. 核心哲学层
- 实事求是：基于事实和数据进行分析
- 矛盾论：识别和解决主要矛盾
- 实践论：理论联系实际
- 群众路线：以用户需求为中心
- 独立自主：培养独立思考能力

### 2. 战略方法层
- 军事战略原则：保存自己，消灭敌人
- 统一战线：团结一切可以团结的力量
- 战略与策略：长期目标与短期行动结合
- 农村包围城市：侧翼突破的战略思想

### 3. 组织建设层
- 民主集中制：民主与集中的辩证统一
- 批评与自我批评：持续改进机制
- 组织建设：结构优化和人才培养
- 自我革命：不断革新和自我超越

### 4. 实践方法层
- 调查研究：深入实际的科学方法
- 群众工作方法：密切联系群众的工作方式
- 典型调查与试点推广：从点到面的工作方法

### 5. 历史案例层
- 古田会议（1929）：建军原则的确立
- 遵义会议（1935）：战略转折的关键
- 延安整风（1942-1945）：思想建设的重要实践
- 重庆谈判（1945）：政治斗争的智慧
- 三大战役（1948-1949）：战略决战的典范
- 农村包围城市实践：革命道路的成功探索

### 6. 现代应用层
- 商业战略应用：企业竞争和发展策略
- 问题解决方法论：系统性思考和解决方案

## 🔧 核心功能

### 5步分析引擎
1. **场景识别**：智能识别问题场景
2. **矛盾分析**：识别主要矛盾和次要矛盾
3. **原则匹配**：匹配相关哲学原则
4. **历史类比**：参考历史先例分析
5. **方案生成**：生成可操作的解决方案

### 3种输出模板
- **A型：问题分析型** - 详细问题拆解和分析报告
- **B型：战略决策型** - 战略规划和决策支持
- **C型：组织管理型** - 组织结构优化和领导力指导

### 场景矩阵
- **15个应用场景**覆盖商业、管理、战略等领域
- **方法论映射**实现智能原则推荐
- **上下文感知分析**基于场景特征定制

## 🚀 快速开始

### 安装与使用

```bash
# 在WorkBuddy环境中加载Skill
/mao-thinking-assistant

# 使用分析引擎
python scripts/contradiction_analysis.py --scenario "strategic_decision" --input "您的问题描述"
```

### 核心分析流程

1. **识别问题场景**：确定适用的应用场景
2. **应用矛盾分析方法**：识别主要矛盾和次要矛盾
3. **匹配相关哲学原则**：选择适用的毛泽东思想原则
4. **参考历史案例类比**：从历史先例中学习经验
5. **生成具体解决方案**：制定可操作的改进措施
